# 2026-04-09

## Summary

- Scaffolded the first Cargo workspace for `ssw-rs`.
- Added `ssw-core`, `ssw-actix`, `ssw-html`, and `ssw-components`.
- Implemented a minimal compileable API surface and verified it with `cargo test`.

## Facts

- `ssw-core` currently defines `Render`, `HtmlKind`, `HtmlResponse`, `TextResponse`, `Redirect`, and `Response`.
- `ssw-html` currently uses a small owned `Markup` type with escaping helpers and simple document composition.
- `ssw-actix` currently exposes response conversion helpers rather than a more ambitious adapter abstraction.
- `ssw-components` currently contains only thin proof-of-shape helpers.

## Risks

- The current HTML layer is intentionally too small to judge long-term ergonomics.
- `Render` may prove too weak or too general once layouts, assets, and partial rendering become more concrete.
- No example app exists yet, so the current API has only unit-test pressure.
